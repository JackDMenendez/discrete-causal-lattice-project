## Summary
Return handoff consuming PM directive `2026-07-04-extensional-core-calibration-boundary`.
The extensional-core / calibration-boundary architecture is **landed on branch
`feature/v0.3.0-peierls-gauge`** (working tree). `core3d` is confirmed constant-free
(the engine emits only extensional quantities — ticks, hops, tokens, counts,
per-tick phase, lattice-unit positions); every physical-unit conversion is now
quarantined behind the new `dcl_core.calibration` submodule (Layer C), which carries
exactly **one** free calibration debt (`tick_seconds`). This satisfies the timing
gate (draw the wall while core3d is still unreleased → free reshape, not a break) and
sets up v0.3.0 to ship the Peierls gauge coupling and the calibration wall in one cut.
